# EACH_SPEC.md

この文書は `00漫画用Camera Position Manager` の個別仕様を、現行実装ベースで整理したものである。
明示的な改良依頼がない限り、本書に記載した UI 構成・名称・基本挙動を維持する。

## 1. アドオンの目的

- 漫画制作時に、コマ画像をカメラの下絵として読み込み、カメラ位置・回転・画角・解像度を合わせながら背景位置を調整するためのアドオン。
- カメラ構図の記録と再呼び出しを行い、複数コマや複数カットの再現をしやすくする。
- 下絵画像の切替、記録ストックの管理、必要に応じたドリーズーム補助を提供する。
- 基本運用は単一カメラ中心とし、複数カメラの切替は Blender 標準機能側で行う前提とする。

## 2. 対応前提

- Blender 5.1 対応を基本とする。
- 可能な限り下位互換も維持する。
- 既存コード上の `bl_info["blender"]` は `2.80.0` であり、下位互換を残しつつ Blender 5.x 系も考慮する。
- UI 表記は日本語を基本とする。

## 3. アドオン名と表示

- アドオン名は `00漫画用Camera Position Manager` とする。
- メインパネルの見出しは `カメラコントロール Ver.x.xxx` 形式とする。
- 3Dビュー右側 N パネルのカテゴリは `カメラ` とする。
- 3Dビュー上部ヘッダーに、`記録` と `画像読込` のショートカットボタンを表示する。

## 4. UI 構成

- UI は 3Dビュー N パネル内の複数 Panel で構成する。
- セクション名と並び順は以下を基準とする。

1. `カメラコントロール Ver.x.xxx`
2. `記録・読込`
3. `追加データ記録`
4. `画像送り`
5. `下絵`
6. `ドリーズーム`
7. `設定他`

### 4.1 カメラコントロール

- アクティブカメラ未設定時は警告を表示する。
- 焦点距離を編集できる。
- ドリーズーム有効中は焦点距離入力を無効化する。
- `カメラをビューに` の切替を表示する。
- カメラ位置 X/Y/Z を直接編集できる。
- 各位置軸にロック切替を表示する。
- 各位置軸に `0` ボタンを表示し、その軸だけをゼロにできる。
- カメラ回転 X/Y/Z を直接編集できる。
- 各回転軸にロック切替を表示する。
- 各回転軸に `0 / 90 / 180 / 270` のスナップボタンを表示する。

### 4.2 記録・読込

- 現在フレームを表示・編集できる。
- 現在カメラに設定中の下絵ファイル名を表示する。
- 出力解像度 X/Y を表示・編集できる。
- `記録` と `削除` ボタンを表示する。
- `現在のカメラへ下絵を読込` と `再読込` ボタンを表示する。
- 保存ストックがある場合、保存済み一覧のプルダウンを表示する。
- 保存ストックがある場合、`<` `>` ボタンで前後のストックへ移動できる。
- ストック総数を表示する。

### 4.3 追加データ記録

- 選択中ストックに紐づく摘要メモを編集できる。
- ストック未保存時は無効表示とする。
- 記録時は摘要メモも保存対象とする。
- `選択OBJデータ` チェックを表示する。
- `選択OBJデータ` が ON の場合、選択中オブジェクト名一覧を表示する。
- 選択中オブジェクト名一覧は、記録済みOBJデータと同程度の幅を持つボタン行として表示する。
- `選択OBJデータ` が ON の場合のみ、選択中オブジェクトの位置・回転・サイズを記録対象へ含める。
- 選択中ストックに記録済みの選択OBJデータがある場合、`記録済みOBJデータ` としてオブジェクト名一覧を表示する。
- `記録済みOBJデータ` のオブジェクト名を押すと、同名オブジェクトを選択する。
- `記録済みOBJデータ` の一覧は、行間を詰めたコンパクトな表示とする。
- `記録済みOBJデータ` のオブジェクト名を Shift+クリックした場合、既存選択を維持して追加選択する。
- `追加データ記録` ボタンを表示する。
- `追加データ記録` は、選択中ストックと現在状態の基本記録データが同一で、摘要メモまたは選択OBJデータなどの追加データだけが異なる場合に限り、選択中ストックへ上書き保存する。
- 上書き条件に当てはまらない場合の `追加データ記録` は、通常の `記録` と同じ挙動とする。

### 4.4 画像送り

- 対象として、現在設定中の読込場所フォルダを表示する。
- `ストック済は読み込まない` を切り替えできる。
- `<` `>` で読込場所フォルダ内の画像を前後切替できる。
- 画像送り実行時は、摘要メモ入力欄をクリアする。
- 対応拡張子は `.png .jpg .jpeg .webp .tif .tiff .bmp` とする。
- 画像順はファイル名の自然順ソートを基準とする。

### 4.5 下絵

- 現在カメラに下絵がない場合は `下絵がありません` と表示する。
- 下絵がある場合、以下を表示する。
  - `表示`
  - `不透明度`
  - `深度`
  - `外枠の濃さ`
- 下絵表示はカメラの Background Images 設定と同期する。

### 4.6 ドリーズーム

- `有効` トグルを持つ。
- ターゲットオブジェクトを指定できる。
- `候補フィルタ` を持つ。
- フィルタ有効時は `含む文字` により候補を絞り込む。
- 有効時は `レンズ最小(mm)` `レンズ最大(mm)` `最小距離` を表示する。
- ドリーズームは正式搭載機能とする。
- ただし本アドオンの主目的である下絵合わせとカメラ位置管理を補完する補助系機能として扱う。
- 有効中は、カメラ移動に応じてターゲットの見かけサイズを維持するようレンズを自動調整する。
- レンズが上下限に達した場合、その先は距離ロックで破綻を抑える。

### 4.7 設定他

- `レンダリング後出力フォルダ開く` を切り替えできる。
- `出力フォルダを開く` ボタンを表示する。
- `カメラデータタブへ移動` ボタンを表示する。
- `設定` トグル展開時に、以下を表示する。
  - `記録データの初期化`
  - `ストックデータを保存`
  - `ストックデータを読込む`
  - `ストックデータ追加読込`
  - `出力フォルダを指定`
  - `読込場所を設定`
  - `保存データを管理`
  - `ソート（降順）`
  - `ソート（昇順）`
  - `全OBJデータ記録`
- `全OBJデータ記録` を ON にすると、全オブジェクト位置・回転・スケールの専用ストックUIを表示する。

## 5. 基本動作仕様

### 5.1 下絵読込

- 下絵読込時、アクティブカメラが無ければカメラを新規作成してシーンカメラに設定する。
- 下絵読込時は可能な限り `ビューにカメラを合わせる` を実行し、その後カメラビューへ切り替える。
- 読み込んだ画像は現在カメラの先頭 Background Image スロットへ設定する。
- 読込直後の下絵深度は `FRONT` とする。
- `FRONT` を基本運用とするが、必要時に他の深度も選択できるよう残す。
- 読込直後は下絵表示を ON にする。
- 画像サイズが取得できる場合、レンダー解像度を画像サイズに合わせる。
- ファイル名末尾 6 桁が数値なら、その値を現在フレームへ設定する。
- 読み込んだ画像の親フォルダを「読込場所」として記憶する。

### 5.2 下絵再読込

- 現在の下絵画像に有効なファイルパスがあれば、その画像を再読込する。
- 現在画像が無いかパス無効の場合、選択中ストックの `bg_image` と読込場所フォルダから再解決を試みる。
- 再読込時も可能なら解像度を画像サイズへ同期する。

### 5.3 記録

- 記録対象は以下とする。
  - カメラ位置
  - カメラ回転
  - 解像度 X
  - 解像度 Y
  - 焦点距離
  - 下絵ファイル名
  - 下絵不透明度
  - 下絵深度
  - 現在フレーム
  - 摘要メモ
  - 選択OBJデータの有効フラグ
  - 選択OBJの名前
  - 選択OBJの位置
  - 選択OBJの回転
  - 選択OBJのサイズ
  - 作成時刻
- 下絵画像そのものではなく、基本的にファイル名のみを保存する。
- `選択OBJデータ` が OFF の場合、選択OBJ情報は保存しない。
- 新規記録時、保存内容と摘要メモの両方が同一の既存ストックがある場合は重複登録しない。
- 保存内容が同一でも摘要メモが異なる場合、`記録` は新規ストックとして追加する。
- `追加データ記録` は、選択中ストックとカメラ位置・カメラ回転・解像度・焦点距離・現在フレーム・下絵ファイル名・下絵不透明度・下絵深度が同一で、摘要メモまたは選択OBJデータなどの追加データだけが異なる場合に限り、選択中ストックを上書きする。
- 上記以外の `追加データ記録` は通常 `記録` と同じく、完全一致ならスキップし、そうでなければ新規ストック追加とする。

### 5.4 ストック読込・適用

- ストック選択時は、保存内容を現在カメラへ即時反映する。
- 適用対象は以下とする。
  - フレーム
  - カメラ位置
  - カメラ回転
  - 解像度 X
  - 解像度 Y
  - 焦点距離
  - 下絵ファイル名から再解決した画像
  - 下絵不透明度
  - 下絵深度
  - 必要時は選択OBJデータとして記録された各オブジェクトの位置・回転・サイズ
- ストック適用後はカメラビューへ切り替える。
- 下絵ファイルが見つからなくても、下絵表示状態と表示設定は可能な範囲で維持する。

### 5.5 ストック削除・管理

- `削除` は現在選択中のストックを削除する。
- `保存データを管理` では、保存ストックを一覧表示し、個別削除できる。
- 一覧では `下絵ファイル名` と `摘要メモ` を確認できる。
- ソートは `bg_image` 名を主キー、`created_at` を副キーとする。

### 5.6 ストック入出力

- ストックデータは JSON で保存する。
- 既定保存先は Blender の user config 配下 `camera_positions.json` とする。
- `ストックデータを保存` の初期ファイル名は、現在の読込場所フォルダの末尾名を使い、`フォルダ名_3D座標データ_[YYYY_MMDD].json` 形式とする。
- JSON の最上位は以下を持つ。
  - `camera_data`
  - `output_folder_path`
  - `background_image_folder_path`
- `ストックデータを読込む` は現在データを置き換える。
- `ストックデータ追加読込` は現在データへ追記する。
- 追加読込時、`完全同一データは登録スキップ` が ON の場合は完全一致ストックを追加しない。

### 5.7 画像送り

- 画像送りは「読込場所」フォルダ内の画像を前後ループ移動する。
- 現在画像が一覧中に存在する場合、その前後へ移動する。
- 現在画像が一覧中に存在しない場合、先頭画像を基準とする。
- `ストック済は読み込まない` が ON の場合、ストック済み下絵ファイル名と同名の画像を候補から除外する。
- 画像送り時も解像度同期と末尾 6 桁によるフレーム同期を行う。
- 画像送り成功時は、前回ストック由来や手入力で残っている摘要メモを誤記録しないよう、摘要メモ欄を空文字へ戻す。

### 5.8 出力フォルダ

- 出力フォルダパスを保持できる。
- 出力フォルダは基本的にレンダー保存先として扱う。
- 運用上は、コンポジットの出力ノードで指定する保存先と同じ場所を指定する前提とする。
- `出力フォルダを開く` で OS のファイルブラウザを開く。
- レンダリング完了時、`レンダリング後出力フォルダ開く` が ON なら Blender ウィンドウを最小化して出力フォルダを開く。

### 5.9 カメラデータタブ移動

- アクティブカメラが無ければ新規作成する。
- 可能な限り安全にカメラを選択し、Properties エリアを `DATA` タブへ切り替える。

### 5.10 ショートカット

- アドオン設定上の案内として以下を表示する。
  - `Insert : 最新のカメラ位置を呼び出し`
  - `Ctrl + Insert : カメラ位置を保存`
  - `Ctrl + Shift + Insert : 下絵を読み込む`
- 参考ショートカットとして登録を試みる構成であるため、この運用は維持対象とする。

### 5.11 全OBJデータ記録

- 全OBJデータ記録は、通常のカメラストックとは別の専用ストックとして扱う。
- 記録対象は、カメラを含む全オブジェクトの名前・位置・回転・スケールとする。
- 全OBJデータの各スロットは、blendファイル名、記録日、摘要メモ、全オブジェクトデータを持つ。
- スロット表示名は `blendファイル名 [YYYY_MMDD]` 形式とする。
- 全OBJデータ専用の摘要メモ欄を持ち、既存の追加データ記録の摘要メモとは別管理とする。
- 記録時、blendファイル名と全オブジェクトデータが完全一致する既存スロットがある場合は、重複登録しない。
- 全OBJデータのスロット一覧では、各スロット名の横に `削除` ボタンを表示する。
- 全OBJデータの個別削除時は、確認ダイアログを表示してから削除する。
- 全OBJデータは既存カメラストックとは別の JSON ファイルとして保存・読込できる。
- 全OBJデータの JSON は `all_object_data` を最上位キーとする。
- 全OBJデータ `読込` 時は、現在操作中の blend ファイル名とスロット内の blend ファイル名が一致する場合のみ適用できる。
- blend ファイル名が一致しない場合は、`操作中のblendファイルとスロットのblendファイルが同名でないと読み込みません` と警告して中止する。
- blend ファイル名が一致する場合でも、適用前に `オブジェクト位置・回転・スケール全データを読み込みますか` と最終確認する。
- 適用時、同名オブジェクトが存在する場合のみ位置・回転・スケールを反映し、存在しないオブジェクトはスキップする。

## 6. データ互換・復旧方針

- 壊れた JSON や旧形式データでも、可能な限り正規化して読み込む。
- 欠損値は既定値で補完する。
- 無効な保存インデックスは `0` に補正する。
- JSON が壊れている場合は空データとして初期化する。
- 保存処理はテンポラリファイル経由の原子的置換を基準とする。

## 7. 改修時の維持ルール

- 既存の UI セクション名は変更しない。
- 既存ボタン名・ラベル名・カテゴリ名は変更しない。
- 既存の記録項目、JSON キー、読込場所記憶、画像サイズ連動、末尾 6 桁フレーム連動は、明示依頼がない限り変更しない。
- 下絵の保存単位は「画像本体」ではなく「ファイル名ベースの再解決」であるため、この前提を勝手に変えない。
- カメラ未設定時に必要なら自動生成して継続できる挙動は維持する。
- ドリーズームは独立補助機能として維持し、通常のカメラ位置記録ワークフローを壊さないこと。
- 内部実装はモジュール化を進めてよいが、責務分離を目的とし、外部仕様や既存ワークフローは変えないこと。
- 少なくとも UI、保存処理、下絵処理、ドリーズーム処理は責務単位で整理してよい。

## 8. 配布・運用確定事項

1. Blender 対応は `5.1 基準 + 可能な限り下位互換維持` とする。
2. 出力フォルダはレンダー保存先として扱い、コンポジット出力ノードと同じ場所を指定する運用を基本とする。
3. 下絵深度は読込直後の既定値を `FRONT` としつつ、選択肢自体は残す。
4. ストック重複判定で `摘要メモ` を無視する現仕様を正とする。
5. ドリーズームは正式搭載機能だが、主目的を補完する補助系機能として扱う。
6. 想定運用は単一カメラ中心とし、複数カメラ切替は Blender 標準側で対応する。
7. 配布 ZIP 名は `00_xxxxx_ver_n_nnn.zip` 形式を採用する。
8. ZIP ファイル名には `00_` 接頭辞を付ける。
9. アドオン本体フォルダ名は、現状で支障がない限り既存名称を維持候補とする。

## 9. 改良前提メモ

- 今後の改修では、まずこの `EACH_SPEC.md` を優先し、未確定事項に関係する変更は事前確認を取る。
- 既存動作の根本に関わる変更は、漫画制作時のカメラ合わせ作業フローを維持できることを前提に判断する。
