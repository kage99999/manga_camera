# SPEC.md

この文書は、Blenderアドオン開発における共通仕様を定義する。

## 1. UI設計の共通仕様

- Blender 5.1 対応を基本にする。可能なら下位互換も保つ
- アドオンのUIは、可能な限りセクションごとに分割すること。
- 各セクションは、可能な限り Blender 標準のネイティブな `Panel` として実装すること。
- 各セクションは、Blender 標準の挙動で折りたたみ可能であること。
- セクションの並び順は、可能な限り Blender 標準のドラッグ＆ドロップによる並べ替えに対応できる構成にすること。
- Blender 標準の `Panel` で実現できる場合は、1つのパネル内に疑似的な箱UIを作ってセクション分けする方式は採用しないこと。

## 2. モジュール構成の共通仕様

- アドオンは、可能な限りモジュール化し、保守性とメンテナンス性を高めること。
- ただし、規模や内容によって単一ファイルの方が安全かつ適切な場合は、その構成を許容する。

## 3. 表示言語の共通仕様

- UI内の表記は、特別な指示がない限り日本語を基本とする。
- 既存アドオンで日本語UIが採用されている場合は、その方針を維持する。

## 4. バージョン表示の共通仕様

- パネルヘッダーには、可能な限りアドオン名とバージョンを並べて1行で表示すること。
- 表記例：`アドオン名 Ver.1.234`

## 5. 保存挙動の共通仕様

- 現在の保存ワークフローは、明示依頼がない限り変更しないこと。
- ファイル名、追加文字列、保存先など、既存の保存仕様を尊重すること。
- 既存の出力挙動は安定して維持すること。

## 6. 配布用ZIP構成の共通仕様

- Blender にそのままインストールできる ZIP 形式を維持すること。
- 配布用 ZIP ファイル名は、原則として `00_xxxxx_ver_n_nnn.zip` 形式とする。  
  - `xxxxx` はアドオン名を表す。  
  - `n_nnn` はバージョン番号を表す。
- 配布用 ZIP のルート直下には、アドオン本体フォルダを1つだけ置くこと。
- ZIP 内のアドオン本体フォルダ名は、原則として `00_xxxxx/` 形式とする。
- ZIP のルート直下に `__init__.py` や個別モジュールファイルを直接置かないこと。

### 構成例

```text
00_xxxxx_ver_1_234.zip
└─ 00_xxxxx/
   ├─ __init__.py
   ├─ aaaaaa.py
   └─ bbbbbb.py
```
